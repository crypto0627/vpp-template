## 充電邏輯說明

### 1. 充電會話啟動條件

充電會話在以下條件**同時滿足**時啟動：

- ✅ 區間**開始時間**的小時數 = 0（即 00:00）
- ✅ 當前日期與上次充電日期不同（新的一天）
- ✅ 充電會話未啟動 且 SOC < 370 kWh

### 2. 為什麼 23:30→00:00 區間不充電？

- 該區間的**開始時間**是 23:30（startHour = 23）
- 不滿足 `startHour === 0` 的條件
- 因此充電會話不會在此區間啟動

### 3. 為什麼 00:00→00:30 區間開始充電？

- 該區間的**開始時間**是 00:00（startHour = 0）✅
- 日期從 2026-03-02 變為 2026-03-03 ✅
- SOC = 200 kWh < 370 kWh ✅
- **所有條件滿足，充電會話啟動！**

### 4. 充電過程

- 充電功率：min(PCS最大功率, 契約容量 - 站場負載)
  - PCS 最大功率 = 100 kW
  - 契約容量限制 = 366.7 kW
  - 站場負載 = 30-45 kW
  - 實際充電功率 ≈ 100 kW
- 充電效率：95%
- 充滿條件：SOC ≥ 370 kWh
